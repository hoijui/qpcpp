108 // NOTE01:
109 // QF::tick() must always run to completion and never preempt itself.
110 // In particular, if QF::tick() runs in an ISR, the ISR is not allowed to
111 // preempt itself. Also, QF::tick() should not be called from two different
112 // ISRs, which potentially could preempt each other.
113 //
114 // NOTE02:
115 // On many CPUs, the interrupt unlocking takes only effect on the next
116 // machine instruction, which happens to be here another interrupt lock.
117 // The assignment of a volatile variable requires a few instructions, which
118 // the compiler cannot optimize away. This ensures that the interrupts get
119 // actually unlocked, so that the interrupt latency stays low.
